# Flipova Studio

Flipova Studio is a **local visual builder** that lets you drag-and-drop layouts, components, and blocks from the Foundation registry, configure props visually, connect data sources, and generate clean React Native code — all from the browser.

## Quick start

```bash
# Install Foundation
npm install @flipova/foundation

# Start the studio
npx flipova-studio
# → http://localhost:4200
```

### Options

| Flag | Default | Description |
|---|---|---|
| `--port <n>` | `4200` | HTTP port for the server and UI |
| `--dev` | — | Start Vite dev server alongside the API server |

```bash
npx flipova-studio --port 3000
npx flipova-studio --dev   # HMR at http://localhost:5173
```

### Docker

```yaml title="docker-compose.yml"
services:
  flipova-studio:
    image: ghcr.io/flipova/foundation:latest
    ports:
      - "4200:4200"
    volumes:
      - studio-data:/app/.flipova-studio
      - ./generated:/app/generated
volumes:
  studio-data:
```

```bash
docker-compose up -d
# → http://localhost:4200
```

## Architecture

```
Browser UI (React + Vite)
       │  REST /api/*
       │  WebSocket ws://localhost:4200/ws
       ▼
Express Server (Node.js)
  ├── GET  /api/registry          → Foundation registries (layouts, components, blocks)
  ├── GET  /api/project           → Load project
  ├── PUT  /api/project           → Save project
  ├── POST /api/pages             → Create page
  ├── PUT  /api/pages/:id         → Update page tree
  ├── DEL  /api/pages/:id         → Delete page
  ├── POST /api/generate          → Generate full project
  └── POST /api/generate/preview/:pageId → Preview one page
       │
       ▼
.flipova-studio/project.json      (persisted project state)
       │  on generate
       ▼
./generated/                      (output React Native app)
  ├── App.tsx
  ├── screens/*.tsx
  ├── navigation/index.tsx
  ├── theme/index.ts
  ├── services/*.ts
  └── hooks/*.ts
```

## UI panels

### Library panel

Browse and search all available items from the Foundation registry:

- **Layouts** — Box, Stack, DashboardLayout, AuthLayout…
- **Components** — Button, TextInput, Avatar, Badge…
- **Blocks** — pre-assembled UI sections

Drag any item onto the **Canvas** to add it to the current page.

### Canvas

The main editing area. Shows a live preview of the current page inside a device frame (iOS / Android / Web). You can:

- Click to select a node
- Drag to reorder nodes
- Double-click to edit text props inline

### Layers panel

Shows the full component tree of the current page. Click any row to select a node. Drag rows to reparent.

### Properties panel

Displays all props for the selected node. Each prop has a type-appropriate input:

| Prop type | Input |
|---|---|
| `string` | Text field |
| `number` | Number stepper |
| `boolean` | Toggle switch |
| `enum` | Dropdown |
| `color` | Color picker + theme swatches |
| `radius` | Radius picker |
| `spacing` | Spacing slider |

**Smart Input** — any prop field can be linked to a data expression:

| Prefix | Source |
|---|---|
| `$state.x` | Page state variable |
| `$query.x` | API query result |
| `$global.x` | Global state |
| `$theme.x` | Theme token |
| `$token.x` | Design token |
| `$nav.x` | Navigation params |
| `$device.x` | Device info (platform, width…) |
| `$node.id.x` | Another node's prop |

### Logic panel

Add **events** and **actions** to any node:

**Events**: `onPress`, `onChange`, `onFocus`, `onBlur`, `onMount`, `onScroll`…

**Actions**:

| Action | What it does |
|---|---|
| `navigate` | Push a new screen |
| `setState` | Update a page state variable |
| `callApi` | Trigger an API query |
| `openModal` / `closeModal` | Toggle modal visibility |
| `alert` | Show a native alert |
| `toast` | Show a toast notification |
| `haptics` | Trigger haptic feedback |
| `clipboard` | Copy text to clipboard |
| `openURL` | Open a URL in the browser |
| `conditional` | Branch on a condition |
| `loop` | Iterate over an array |
| `parallel` | Run multiple actions concurrently |
| `delay` | Wait N milliseconds |
| `custom` | Insert raw TypeScript code |

### Code panel

Shows the live-generated `.tsx` and `.hook.ts` code for the current page. Copy to clipboard or run a full generation.

## Code generation

### Generate a full project

```bash
npx flipova-studio generate
```

Output structure:

```
generated/
├── App.tsx                    # Root app component + navigation
├── screens/
│   ├── HomeScreen.tsx         # Generated screen component
│   └── HomeScreen.hook.ts     # Generated hook (state, API calls, handlers)
├── navigation/
│   └── index.tsx
├── theme/
│   └── index.ts
├── services/
│   └── api.ts
├── hooks/
│   └── useQueryName.ts        # Auto-generated query hooks
├── controllers/
│   └── useServiceController.ts
├── package.json
└── tsconfig.json
```

### Generated code rules

The generator enforces these rules:

1. **No `require()`** — ESM-only imports
2. **All logic in hooks** — screen files are pure JSX, hooks hold all state and handlers
3. **Typed props** — every generated component has typed prop interfaces
4. **Safe URLs** — `openURL` actions use an `isSafeUrl()` guard

### Preview one page

```bash
# Via CLI
npx flipova-studio generate --page home

# Via REST
curl -X POST http://localhost:4200/api/generate/preview/page-id-here
```

## WebSocket events

Connect to `ws://localhost:4200/ws` for real-time updates:

| Event | Payload | When |
|---|---|---|
| `page:created` | `PageDocument` | New page added |
| `page:updated` | `PageDocument` | Tree or props changed |
| `page:deleted` | `{ id }` | Page removed |
| `theme:changed` | `{ theme: string }` | Active theme switched |
| `project:saved` | `ProjectDocument` | Full project saved |

## Project file

The Studio persists your project in `.flipova-studio/project.json`. Commit this file to version control so your team can share the same project state.

```json title=".flipova-studio/project.json (excerpt)"
{
  "name": "My App",
  "theme": "light",
  "pages": [
    {
      "id": "p_home",
      "name": "Home Screen",
      "route": "home",
      "root": { "id": "n_root", "kind": "layout", "registryId": "RootLayout", ... }
    }
  ],
  "services": [],
  "navigation": { "type": "stack", "screens": [] }
}
```

## Adding Foundation components to the registry

When you add a new component to Foundation, it automatically appears in the Studio Library as long as it's registered in `foundation/layout/registry/components.ts`. See the [Contributing guide](../contributing) for step-by-step instructions.

## Troubleshooting

| Symptom | Fix |
|---|---|
| Studio won't start | Check if port 4200 is in use: `npx flipova-studio --port 3001` |
| Components missing from Library | Rebuild Foundation: `npm run build`, then restart |
| Code generation fails | Open the browser console for the detailed error; check that all required props are set |
| WebSocket disconnects | Refresh the page; the server reconnects automatically |
