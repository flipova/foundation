# Theming

Foundation's theme system is built around a **ColorScheme** — a flat object of semantic color tokens that is swapped at runtime. Every component reads its colors from the active scheme via `useTheme()`.

## Built-in themes

| Key | Description |
|---|---|
| `light` | Clean, modern light theme (default) |
| `dark` | Dark-mode theme |
| `neon` | Purple/cyan futuristic theme |
| `spring` | Pink/green fresh theme |
| `summer` | Amber/orange warm theme |
| `autumn` | Deep brown/rust warm theme |
| `winter` | Slate/blue cool theme |
| `halloween` | Violet/green dark theme |
| `christmas` | Green/red festive theme |

---

## Setup

### React Native

```tsx title="App.tsx"
import { FoundationProvider, defineConfig } from '@flipova/foundation/config';

const config = defineConfig({ defaultTheme: 'dark' });

export default function App() {
  return (
    <FoundationProvider config={config}>
      <YourApp />
    </FoundationProvider>
  );
}
```

### React Web

```tsx title="App.tsx"
import { ThemeProvider } from '@flipova/foundation/web';

export default function App() {
  return (
    <ThemeProvider defaultTheme="dark">
      <YourApp />
    </ThemeProvider>
  );
}
```

## Reading the active theme

```tsx
import { useTheme } from '@flipova/foundation'; // or /web

function Card() {
  const { theme, mode, setTheme, availableThemes } = useTheme();

  return (
    <div style={{ background: theme.card, color: theme.foreground }}>
      <p>Active theme: {mode}</p>
    </div>
  );
}
```

### `useTheme()` return value

| Property | Type | Description |
|---|---|---|
| `theme` | `ColorScheme` | Active color scheme object |
| `mode` | `string` | Active theme key, e.g. `"dark"` |
| `setTheme(key)` | `(key: string) => void` | Switch to another theme |
| `availableThemes` | `ThemeRegistry` | All registered themes |

## ColorScheme keys

Every theme must provide these semantic colors:

```ts
type ColorScheme = {
  background:            string;  // main page background
  foreground:            string;  // main text color
  card:                  string;  // card/surface background
  cardForeground:        string;  // text on cards
  primary:               string;  // primary brand color
  primaryForeground:     string;  // text on primary bg
  secondary:             string;  // secondary color
  secondaryForeground:   string;
  muted:                 string;  // subtle background
  mutedForeground:       string;  // subtle text
  accent:                string;  // highlight color
  accentForeground:      string;
  destructive:           string;  // danger / error
  destructiveForeground: string;
  border:                string;  // dividers and outlines
  input:                 string;  // input backgrounds
  ring:                  string;  // focus ring
  success:               string;
  warning:               string;
  error:                 string;
  info:                  string;
  transparent:           string;
  gradients:             ThemeGradients;
};
```

## Switching themes at runtime

```tsx
import { useTheme } from '@flipova/foundation/web';

function ThemeSelector() {
  const { mode, setTheme, availableThemes } = useTheme();
  const keys = Object.keys(availableThemes);

  return (
    <div style={{ display: 'flex', gap: 8 }}>
      {keys.map((key) => (
        <button
          key={key}
          onClick={() => setTheme(key)}
          style={{
            fontWeight: mode === key ? 700 : 400,
            textDecoration: mode === key ? 'underline' : 'none',
          }}
        >
          {key}
        </button>
      ))}
    </div>
  );
}
```

## System color scheme

`useColorScheme()` detects the system preference and updates live:

- **React Native**: uses `useColorScheme` from `react-native`
- **Web** (`/web` entry): uses `window.matchMedia('(prefers-color-scheme: dark)')` with a live listener

```tsx
import { useColorScheme } from '@flipova/foundation/theme';

function SystemAware() {
  const scheme = useColorScheme(); // 'light' | 'dark'
  return <p>System prefers: {scheme}</p>;
}
```

To follow the system automatically, set `defaultTheme` to the result of `useColorScheme()`:

```tsx
import { useColorScheme } from '@flipova/foundation/theme';
import { FoundationProvider, defineConfig } from '@flipova/foundation/config';

function App() {
  const scheme = useColorScheme();
  const config = defineConfig({ defaultTheme: scheme });
  return <FoundationProvider config={config}><YourApp /></FoundationProvider>;
}
```

## Persisting the user's theme

```tsx
import { useTheme } from '@flipova/foundation/web';
import { useEffect } from 'react';

function ThemePersistence() {
  const { mode, setTheme } = useTheme();

  // Load on mount
  useEffect(() => {
    const saved = localStorage.getItem('flipova-theme');
    if (saved) setTheme(saved);
  }, []);

  // Save on change
  useEffect(() => {
    localStorage.setItem('flipova-theme', mode);
  }, [mode]);

  return null;
}
```

## Importing built-in themes directly

```tsx
import {
  lightTheme,
  darkTheme,
  neonTheme,
  springTheme,
  summerTheme,
  autumnTheme,
  winterTheme,
  halloweenTheme,
  christmasTheme,
} from '@flipova/foundation/theme';

// Use as a starting point for a custom theme
const myTheme = {
  ...darkTheme,
  colors: { ...darkTheme, primary: '#ff4488' },
};
```

## Best practices

- **Never hardcode hex values** in component styles — use `theme.primary`, `theme.card`, etc.
- **Provide a theme toggle** so users can choose dark/light mode.
- **Test all 9 built-in themes** before shipping custom components.
- **Use semantic names** (`theme.muted`) rather than descriptive ones (`theme.gray200`).
